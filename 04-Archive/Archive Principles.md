# Archive Principles

## What Belongs in the Archive
The archive contains items that are no longer active but may be valuable for reference, historical context, or potential future use.

### Categories for Archiving
- **Completed Projects** - Finished projects with lessons learned
- **Inactive Areas** - Responsibilities no longer maintained
- **Reference Materials** - Useful information not currently needed
- **Old Versions** - Superseded documents or notes
- **Templates & Examples** - Boilerplates for future use
- **Research Leftovers** - Notes from completed research

## Archiving Process
### When to Archive
1. Project marked as completed/cancelled
2. Area responsibility ended or changed significantly
3. Information is outdated but potentially useful
4. Completing a learning unit or course
5. End of a time period (monthly/yearly review)

### How to Archive
1. **Review** - Ensure all actionable items are processed
2. **Consolidate** - Combine related notes if appropriate
3. **Tag** - Add #archive tag and relevant context tags
4. **Link** - Create backward links from related active notes
5. **Move** - Place in appropriate archive subfolder
6. **Document** - Add brief context note if needed

### Archive Folder Structure
```

├── Projects/
│   ├── Completed/
│   │   ├── Project Name (YYYY-MM-DD)
│   │   └── ...

```

## Retrieval & Usage
### Finding Archived Items
- Use search with `path:04-Archive` or `-path:04-Archive` to exclude
- Browse folder structure for context
- Use tags like #archive, #project/name, #topic/area
- Check graph view for orphaned notes that might be archived

### When to Retrieve
- Reference for similar new projects
- Legal or compliance requirements
- Historical analysis or trend tracking
- Revisiting past decisions
- Template reuse

### When to Delete
- Information is obsolete or incorrect
- Legal retention period expired
- Multiple copies exist elsewhere
- No reference value after review period
- Storage concerns (rare with text notes)

## Best Practices
### Do's
- Write a brief summary when archiving complex items
- Maintain links to related active notes
- Use consistent naming: `[Project Name] (YYYY-MM-DD)`
- Archive at natural completion points
- Review archive quarterly for relevance

### Don'ts
- Archive items that still have actionable tasks
- Forget to update links if moving notes later
- Archive without processing (creates digital landfill)
- Use archive as a procrastination tool
- Over-organize; simple structure is better

## Maintenance Schedule
- **Monthly**: Quick review of new archive additions
- **Quarterly**: Review relevance, consider deletion of obsolete items
- **Yearly**: Deep archive review, update folder structure if needed
- **Ongoing**: Add backward links when creating new related content
