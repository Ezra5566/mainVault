# Launch Checklist

> **Pre‑launch** and **Launch** day tasks for a new product or service.

## Pre‑Launch (Weeks ‑2 → ‑1)
- [ ] Finalize **MVP** features & UI.  
- [ ] Create **marketing assets** (landing page, demo video).  
- [ ] Set up **email capture** and automation flow.  
- [ ] Load **pricing** and payment gateway (Stripe, PayPal).  
- [ ] Prepare **support docs** (FAQ, knowledge base).  
- [ ] Conduct **final QA** (bugs, security).  
- [ ] Schedule **launch announcement** (date, channels).  

## Launch Day (Day 0)
- [ ] Publish **landing page** live.  
- [ ] Send **announcement email** to list.  
- [ ] Publish **social posts** (Twitter, LinkedIn, Facebook).  
- [ ] Monitor **traffic & sign‑ups** (real‑time dashboard).  
- [ ] Handle **first support tickets** promptly.  

## Post‑Launch (Day +1 → +7)
- [ ] Review **KPIs** (sign‑ups, churn, NPS).  
- [ ] Iterate on **feedback** (bug fixes, UX tweaks).  
- [ ] Initiate **paid ad campaigns**.  
- [ ] Schedule **webinar** or live demo (Day +3).
